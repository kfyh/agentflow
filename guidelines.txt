### Agent Execution & Safety Guidelines (CRITICAL)
To prevent command timeouts and shell hangs inside the container environment, adhere to the following rules:
1. **No Foreground Web Servers**: Do NOT start dev servers (e.g., `npm run dev` or webpack dev server) or file-watchers in the foreground. If you must verify them, start them in the background (e.g. `npm run dev &` or `nohup`), check the endpoint with `curl` or `wget`, and immediately terminate the process.
2. **Do Not Compile Rust/Tauri inside the Container**: Rust compilation inside a double-virtualized container (Colima on macOS) is highly resource-intensive and will time out. Do NOT run `npm run tauri build` or Cargo compile commands. Focus on writing the configuration files, Rust source files, and web assets, and verify them using fast static checks. Leave binary compilation to the host.
3. **Use Fast, Non-Blocking Verification**: Verify project setup using `tsc --noEmit` (type checking) and `npm run lint` (ESLint analysis) which execute in seconds and exit cleanly.
4. **Track Progress with a Tasklist**: At the start of the project, create a `tasklist.md` file in the root of the workspace (`/workspace/tasklist.md`). Break down the requirements into clear tasks and subtasks. Update the file dynamically, marking items as in-progress or completed (`[x]`) as you make progress. This ensures that if the agent session restarts or times out, the next agent can read this file and resume work exactly where you left off.
5. **Write Outputs to Workspace (No .gemini Pollution)**: Any requested outputs, reports, documentation, or code files you generate must be written directly inside the active project workspace directory (or its subfolders). Do NOT save outputs, reports, or results in the agent's private directories, `.gemini/` app data folders, `/tmp`, or home directories. All user-facing deliverables must reside in the workspace so they are easily accessible and do not pollute system configuration directories.
6. **No Git Available**: Git is intentionally not installed in this container environment. Do NOT run any `git` commands (e.g., `git status`, `git diff`, `git commit`, `git clone`, etc.). All version control tasks must be left to the host machine.
7. **No NPM Registry Keys (.npmrc)**: No private npm registry keys or `.npmrc` files are configured in the container. As a result, you cannot access or install private scoped packages, specifically under namespaces like `@aristocrattechnologiesinc` and `@roxorgaming`. Do not run package manager install commands (e.g., `npm install`, `yarn`) that depend on these private registry tokens.
8. **No Sudo/Root Privilege Escalation**: Sudo/root permissions are not available or allowed in this container environment. Do NOT run any commands prefixed with `sudo` (e.g., `sudo apt-get`, `sudo npm install`), do not attempt to escalate privileges, and do not request root access. All package installations or modifications must be executed with standard user permissions.
